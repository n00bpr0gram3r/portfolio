3:T57a,<h1>DNS Rebinding in Local HTTP/SSE-based MCP Server</h1>
<h2>Vulnerability Summary</h2>
<p>Model Context Protocol (MCP) servers utilizing HTTP/SSE transports (running locally on port <code>6333</code> or similar) were found to lack Host header validation and authentication. This enables external malicious sites to perform a DNS Rebinding attack to run arbitrary local tools.</p>
<h2>Technical Breakdown</h2>
<p>When a user visits a malicious website, the page runs JavaScript that connects back to the attacker&#39;s domain.</p>
<ol>
<li>The attacker&#39;s DNS server answers with a low TTL (1 second), mapping the domain to the attacker&#39;s server.</li>
<li>The page loads and executes.</li>
<li>Once the TTL expires, a second request triggers a new DNS query. The attacker&#39;s DNS server now answers with <code>127.0.0.1</code>.</li>
<li>The browser, thinking the site is still the same origin, sends HTTP requests to the local port, bypassing the Same-Origin Policy.</li>
</ol>
<h2>Remediation &amp; Hardening</h2>
<ol>
<li><strong>Validate Host Headers:</strong> Configure HTTP middleware to verify that all incoming requests contain a Host header explicitly matching <code>localhost</code> or <code>127.0.0.1</code>.</li>
<li><strong>Transition to Stdio Transport:</strong> Standardize on standard input/output (stdio) pipelines rather than running local network port listeners.</li>
</ol>
